# Procurement Reporting (SKKtoSDUv2)

## Статус

⚠️ Текущая версия SQL-скриптов является BASELINE (v1)

- Получены от базистов
- Не являются подтвержденным источником истины
- Требуют верификации на соответствие методике

## Состав

1. CLASSIFIER — определение категории договора
2. DEFINITELY_COMPLEX — обработка цепочек допников
3. DEFINITELY_SIMPLE — договора без допников
4. DIRECTORY — справочник для витрины

## Важно

Данный репозиторий используется для:

- анализа корректности SQL
- трассировки логики (методика → SQL → данные)
- выявления ошибок и аномалий

## Следующие шаги

- разметка RULE_ID
- сопоставление с методикой
- выявление расхождений
